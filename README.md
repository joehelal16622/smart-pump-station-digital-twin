# Smart Pump Station Digital Twin with ML Fault Detection and Energy Optimization

A Streamlit-based mechanical engineering project that simulates a pump station, generates synthetic sensor data, detects operational faults using machine learning, and recommends optimized pump settings to reduce energy cost.

**Live App:** https://smart-pump-station-digital-twin-qrtfautxmaktskt3appv6xh.streamlit.app/

## Project Overview

This project models a pump station feeding a pipeline system. The dashboard allows users to adjust operating conditions such as pipe geometry, pump speed, valve opening, elevation gain, target demand, and fault severity.

The app combines physics-based simulation, synthetic sensor generation, rule-based diagnostics, machine learning classification, and energy optimization into one interactive engineering dashboard.

## Key Features

* Pump and pipeline digital twin
* Pump curve vs system curve visualization
* Flow rate, head, pressure loss, efficiency, power, and energy cost calculations
* Synthetic sensor data for flow, pressure, vibration, motor temperature, power, and efficiency
* Fault injection for:

  * Leak
  * Blockage
  * Bearing wear
  * Impeller fouling
  * Sensor drift
* Rule-based fault diagnosis
* Random Forest machine learning fault classifier
* ML confidence score and fault probability chart
* Confusion matrix and feature importance analysis
* Energy optimization for pump RPM and valve opening
* Recommended operating point to reduce energy cost while meeting demand

## Machine Learning Layer

The app trains a Random Forest classifier on synthetic pump-station cases generated by the digital twin. The model uses sensor statistics and operating-point features to classify the system condition.

Example fault classes include healthy operation, leaks, blockages, bearing wear, impeller fouling, and sensor drift.

The classifier is intended as a portfolio demonstration of how simulated engineering data can be used to build a predictive maintenance workflow.

## Energy Optimization Layer

The optimizer searches different pump RPM and valve-opening combinations. It recommends the operating point that best balances:

* meeting target flow demand
* reducing daily energy cost
* maintaining acceptable pump efficiency
* avoiding high cavitation risk
* avoiding excessive pipe velocity
* reducing unnecessary valve throttling

## Important Note

The machine learning model is trained and tested on synthetic digital-twin data. High accuracy is expected because the simulated fault signatures are controlled. Real-world deployment would require validation using actual pump-station sensor data.
